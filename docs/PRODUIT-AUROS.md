# AUROS — Ce que fait le site

## Mission (une phrase)

**Démocratiser la préparation d’un dossier RWA** : score, data room, admission plateforme, studio réglementaire indicatif — pour qu’un porteur (ou son conseil) arrive crédible sans être expert blockchain.

On **ne** déploie **pas** les smart contracts. On structure l’amont ; plateformes et conseils font l’émission.

## Modèle business (vision)

| Priorité | Objectif |
|----------|----------|
| **Cœur** | Outil + confiance → dossiers qualifiés |
| **Croissance** | Pages « comment tokeniser [actif] », contenu, tunnels SEO |
| **Monétisation cible** | **Petites plateformes RWA** : clients dossier-ready (#1) puis **Liquidity Bridge** (#2) · **Apporteurs** : % sur actif amené |

Détail : `docs/MODELE-ECONOMIQUE.md`

## Ce que le site fait aujourd’hui

| Étape | Utilisateur | Produit |
|-------|-------------|---------|
| 1. Découverte | Visiteur | Score rapide + promesse claire (préparation, pas on-chain) |
| 2. Qualification | Wizard (complet ou express) | Actif, data room 15 docs, conformité, plateforme cible |
| 3. Dossier | Émetteur | Admission %, studio, IA, PDF, data room upload |
| 4. Action | Compte connecté | Sauvegarde cloud, partage, revue AUROS (concierge) |
| 5. B2B | Partenaire | Formulaire `/partners` + annuaire prestataires |

## Ce que le site ne fait pas (volontairement)

- Déploiement smart contract
- KYC Sumsub/Onfido intégré
- Soumission automatique plateforme (revue humaine AUROS)
- Calcul / paiement automatique des commissions partenaires (roadmap)
- Marché secondaire

## Roadmap produit (ordre logique)

1. **Infra prod** — `docs/PROD-LAUNCH.md`
2. **Trafic** — landing par type d’actif + contenu éditorial
3. **Partenaires** — attribution `partner_code` sur leads/dossiers → reporting
4. **Automatisation contenu** — guides, comparatifs, emails
5. **Uniquement si pivot** — KYC, deploy testnet, etc.

## Différenciation

- **Taux d’admission plateforme** (pas un score marketing seul)
- **Data room 5×15** alignée industrie
- **Studio** réglementaire + tokenomics + roadmap dans un même espace
- **UX** : 3 priorités max, parcours express, ton pro sans surcharge
- **Transparence** : indicatif vs exécution on-chain
